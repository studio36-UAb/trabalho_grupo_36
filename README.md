## Estrutura do projeto

Estrutura do projeto a definir pelo grupo, de acordo com a evolução do desenvolvimento.

## Workflow

O projeto utiliza duas branches principais:

- **main** — versão estável e final do projeto
- **develop** — branch de integração e desenvolvimento

Além destas, cada elemento do grupo deverá trabalhar na sua **branch individual**, de forma a evitar conflitos e facilitar a organização do trabalho.

### Regras de trabalho

- Não devem ser feitas alterações diretamente na branch **main**.
- Cada elemento deve desenvolver o seu trabalho na respetiva **branch individual**.
- As alterações concluídas devem ser integradas na branch **develop**.
- Antes de integrar alterações, deve verificar-se se o projeto compila e funciona corretamente no **Visual Studio**.
- Depois de testadas e validadas na **develop**, as alterações poderão ser integradas na **main**.

branch individual -> develop -> main
